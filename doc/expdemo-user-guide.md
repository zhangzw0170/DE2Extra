## 1. 快速开始

### 进入 expdemo

在 shell 命令行中输入：

```
expdemo
```

进入后将看到 expdemo Home 页面，列出所有可用实验。

### 基本流程

```
输入 expdemo → Home 页面 → 输入实验编号 + Enter → 进入实验 → 操作 SW/KEY 观察板载输出
                                                           → q 返回 Home → q 退出到 shell
```

### 三十秒快速体验

1. 输入 `expdemo` 进入
2. 按 `1` + Enter → Exp1 译码器实验
3. 拨动 SW[2:0]，观察 LEDR[7:0] 的译码输出变化
4. 按 `q` 返回主页，再按 `q` 退出

## 2. 导航操作

### Home 页面操作

| 操作 | 串口/键盘 | IR 遥控器 |
|------|-----------|-----------|
| 输入实验编号 | 数字键 `0`-`9` | 数字键 `0`-`9` |
| 确认进入实验 | `Enter` | `RETURN` 或 `PLAY` |
| 浏览下一个实验 | `+` 或 `=` | `CH+` |
| 浏览上一个实验 | `-` | `CH-` |
| 删除已输入编号 | `Backspace` | — |
| 退出到 shell | `q` / `Esc` | `MENU` |

编号输入规则：
- 连续输入数字时自动拼接（如按 `1` → `3` = 13），若超过 13 则以最后一位为新编号
- 未输入编号时按 Enter，默认进入上次选中的实验

### 实验页面操作

| 操作 | 说明 |
|------|------|
| `q` / `Esc` / IR `MENU` | 返回 expdemo Home 页面 |
| 数字键 + Enter | 直接切换到另一个实验（无需先回 Home） |
| 板载 SW / KEY | 操作实验本体（VGA/UART monitor 同步显示输入状态） |

### KEY0 规则

**KEY0 永远是整板硬复位**，不会被传递给任何实验。原实验中的 KEY0 按键统一顺延到 KEY1。

## 3. Monitor 实时监视

进入任何实验后，VGA / UART 终端下半部分显示实时 Monitor：

```
Live Monitor
------------
HW channel : 1  STATUS=0x00000011
SW[17:0]   : 000000000000000101
KEY[3:1]   : 111  (KEY0 = board reset)
Last IR    : -- (none)
IR RAW     : 0  DBG V/CMD: 0 / 0x00
IR REG     : DATA=0x00000000  ST=0x00000000
Uptime     : 0x00000042 s
```

Monitor 字段说明：

| 字段 | 含义 |
|------|------|
| HW channel | 当前硬件通道号（即实验编号） |
| STATUS | 状态寄存器：`[0]=active`, `[7:4]=channel` |
| SW[17:0] | 拨码开关实时值（`1`=ON/上拨） |
| KEY[3:1] | 按键状态（`0`=按下，`1`=松开） |
| Last IR | 最近一次 IR 遥控器命令码 + 名称 |
| IR RAW | IrDA 接收器原始信号电平 |
| DBG V/CMD | IR 调试标志：Valid / Command |
| IR REG | IR 控制器寄存器：DATA / STATUS |
| Uptime | 系统运行时间（秒） |

**注意**：Monitor 只反映 CPU 可观测的输入寄存器，不伪造板载 HEX/LEDR/LEDG/LCD 的回读值。

## 5. 实验详解

### Exp1 — 3-8 译码器

**通道号**：1

**实验内容**：实现标准 3-8 译码器（类似 74LS138），3 位输入经使能门控制后输出 8 位低电平有效信号。

#### 输入

| 拨码开关 | 功能 | 说明 |
|----------|------|------|
| SW[2:0] | A / B / C | 3 位地址输入 |
| SW5 | G1 | 高电平有效使能 |
| SW4 | G2A | 低电平有效使能 |
| SW3 | G2B | 低电平有效使能 |

#### 按键

| 按键 | 功能 |
|------|------|
| KEY0 | 板级硬复位（保留） |
| KEY1-3 | 未使用 |

#### 输出

| 外设 | 显示内容 |
|------|---------|
| LEDR[7:0] | 译码输出（低电平有效），当 G1=1 且 G2A=G2B=0 时正常译码；否则全灭 |

#### 操作示例

1. 拨 SW5=ON（G1=1）、SW4=OFF（G2A=0）、SW3=OFF（G2B=0）使能译码器
2. 拨 SW[2:0]=101（即 5），观察 LEDR5 亮（低电平有效 = LEDR5=0→灭，其余亮）
3. 将 SW4 拨为 ON → 所有输出无效（LEDR 全灭）

### Exp3 — 数码管扫描

**通道号**：3

**实验内容**：四子模式实验，涵盖 7 段译码器、静态显示、动态扫描和数字时钟。

#### 输入

| 拨码开关 | 功能 | 子模式 |
|----------|------|--------|
| SW[17:16] | 子模式选择 | `00`=seg_decoder, `01`=hello_display, `10`=hex_scan, `11`=digital_clock |
| SW[15:14] | 数字时钟额外控制 | 仅 digital_clock 模式 |
| SW[7:0] | 数据输入 | 各模式含义不同 |

#### 按键（仅 digital_clock 模式有效）

| 按键 | digital_clock 功能 | 其他子模式 |
|------|-------------------|-----------|
| KEY0 | 板级硬复位（保留） | 板级硬复位 |
| KEY1 | 时钟复位 | 未使用 |
| KEY2 | 写入分钟值 | 未使用 |
| KEY3 | 写入小时值 | 未使用 |

> **注意**：原实验中 KEY0 的"写秒"功能在 expdemo 中已删除。

#### 输出

| 外设 | 显示内容 |
|------|---------|
| HEX[7:0] | 时钟显示 / 扫描显示（取决于子模式） |
| LEDR | 回显开关状态 |

#### 操作示例 — 数字时钟

1. 拨 SW[17:16]=11 进入 digital_clock 模式
2. SW[7:0] 设定小时/分钟值
3. KEY3 写小时，KEY2 写分钟
4. KEY1 复位时钟

### Exp4 — 双端口 RAM

**通道号**：4

**实验内容**：32×8 双端口 RAM 读写实验，可指定地址进行读/写操作。

#### 输入

| 拨码开关 | 功能 |
|----------|------|
| SW17 | 读写模式（1=写，0=读） |
| SW[15:11] | 5 位地址 |
| SW[7:0] | 8 位数据 |

#### 按键

| 按键 | 功能 |
|------|------|
| KEY0 | 板级硬复位（保留） |
| KEY1 | 写模式下触发一次写入脉冲 |

#### 输出

| 外设 | 显示内容 |
|------|---------|
| HEX | 地址 + 数据（HEX 显示地址高位、地址低位、数据） |
| LEDR | SW 镜像（便于确认拨码状态） |
| LEDG | 读出数据（读模式下 RAM 输出） |
| LEDG8 | 写确认脉冲指示 |

#### 操作示例

1. **写入**：SW17=ON（写模式），SW[15:11]=00010（地址 2），SW[7:0]=10101010（数据 0xAA），按 KEY1 写入
2. **读取**：SW17=OFF（读模式），SW[15:11]=00010（地址 2），观察 LEDG 显示之前写入的 0xAA

### Exp9 — UART 收发

**通道号**：9

**实验内容**：UART 串口收发实验。内部实现 loopback（TX 回环到 RX），发送的字节会被自己接收回来。

#### 输入

| 拨码开关 | 功能 |
|----------|------|
| SW[7:0] | 待发送的 8 位字节 |

#### 按键

| 按键 | 功能 |
|------|------|
| KEY0 | 板级硬复位（保留） |
| KEY1 | 触发一次发送（将 SW[7:0] 的值发送出去） |

#### 输出

| 外设 | 显示内容 |
|------|---------|
| HEX[5:4] | 最近接收到的字节 |
| HEX[1:0] | 最近发送的字节 |
| LEDR | RX 状态指示 |
| LEDG | TX 状态指示 |

#### VGA/UART Monitor 特殊页面

Exp9 有专属的 VGA 监视页面，额外显示：

- **Virtual UART**：说明 TX loopback 机制
- **TX 发送字节**：当前 SW[7:0] 值 + ASCII 字符
- **Last loopback**：最近一次发送/接收的十六进制 + ASCII
- **RX History**：最近 16 次接收字节的 HEX + ASCII 窗口

#### 操作示例

1. 拨 SW[7:0]=01000001（0x41 = 'A'）
2. 按 KEY1 发送
3. 观察 HEX 显示 TX=41、RX=41（loopback 回来）
4. VGA Monitor 的 RX History 累积显示所有发送过的字节

### Exp11 — DDS 正弦波

**通道号**：11

**实验内容**：直接数字频率合成器（DDS），产生可调频率的正弦波。输出为 10 位数字幅度（LEDR 显示或 SignalTap 抓取）。

#### 输入

| 拨码开关 | 功能 |
|----------|------|
| SW[7:0] | 频率控制字 fword |
| SW8 | 模式切换（0/1） |
| SW17 | 时钟模式（0=LED 慢速观察，1=SignalTap 快速捕获） |

#### 按键

| 按键 | 功能 |
|------|------|
| KEY0 | 板级硬复位（保留） |
| KEY1 | DDS 相位累加器复位 |

#### 输出

| 外设 | 显示内容 |
|------|---------|
| LEDG[9:0] | 正弦波幅度采样值（10 位） |

#### VGA Monitor 特殊页面

Exp11 有专属的 VGA 监视页面，额外显示：

- **SW17 模式**：LED 观察（慢速 100kHz）/ SignalTap 捕获（快速 50MHz）
- **SW[7:0] fword**：频率控制字（十六进制 + 十进制）
- **Estimated f0**：估算输出频率
- **Formula**：频率计算公式

#### 频率计算

| 模式 | 公式 |
|------|------|
| SW17=0（LED 慢速） | f₀ = 100kHz × fword / 2²⁰ |
| SW17=1（SignalTap 快速） | f₀ = 50MHz × (fword << 8) / 2²⁰ |

#### 操作示例

1. 拨 SW[7:0]=00000001（fword=1），观察极低频 LEDG 变化
2. 拨 SW[7:0]=10000000（fword=128），观察中频波形
3. SW17=1 + SignalTap 抓取高速波形数据

### Exp12 — 简易 CPU

**通道号**：12

**实验内容**：8 位简易 CPU，支持 5 条指令（ADD / STO / LOD / JMP / JNG），256 字节内存空间。可单步执行或自动运行，实时观察 PC、AC、IR 寄存器和 FSM 状态。

#### 指令集

| 操作码 | 助记符 | 功能 |
|--------|--------|------|
| 0x00 | ADD addr | AC ← AC + MEM[addr] |
| 0x01 | STO addr | MEM[addr] ← AC |
| 0x02 | LOD addr | AC ← MEM[addr] |
| 0x03 | JMP addr | PC ← addr |
| 0x04 | JNG addr | if AC < 0 then PC ← addr |

指令编码：高 8 位 = 操作码，低 8 位 = 地址。如 `0x0305` = JMP 0x05。

#### 输入

| 拨码开关 | 功能 |
|----------|------|
| SW16 | LCD 详细模式开关（1=显示所有 FSM 状态，0=仅显示关键状态） |

#### 按键

| 按键 | 功能 |
|------|------|
| KEY0 | 板级硬复位（保留） |
| KEY1 | CPU 主复位 |
| KEY2 | 单步执行一个 FSM 状态 |
| KEY3 | 自动/手动模式切换 |

#### 输出

| 外设 | 显示内容 |
|------|---------|
| HEX | IR / PC / AC 值 |
| LEDR | 模式指示 + AC 二进制 |
| LEDG | 步数计数器 |
| LCD | 反汇编 + FSM 状态描述（双行 16×2） |

#### VGA Monitor 特殊页面

Exp12 有专属的 VGA 监视页面，额外显示：

- **CPU Snapshot**：PC、AC、IR、FSM 状态、运行模式（AUTO/STEP）、详细模式（FULL/BRIEF）、步数
- **LCD Trace**：最近 8 条 LCD 显示内容的快照历史（包括反汇编行和 FSM 状态行）

#### FSM 状态含义

| FSM | 含义 |
|-----|------|
| 0 | MAR ← PC（取指阶段：送 PC 到地址寄存器） |
| 1 | MDR ← MEM（读存储器） |
| 2 | IR ← MDR, PC++（指令寄存器加载，PC 自增） |
| 3 | DECODE（译码） |
| 4 | MAR ← addr（ADD/LOD/JMP/JNG：取操作数地址） |
| 5 | MDR ← MEM（读操作数） |
| 6 | AC ← AC + MDR（ADD：累加） |
| 7 | MAR ← addr（STO：存数地址） |
| 8 | MEM ← AC（STO：写入存储器） |
| 9 | MAR ← addr（LOD/JMP/JNG：地址） |
| 10 | MDR ← MEM（LOD：读取数据） |
| 11 | AC ← MDR（LOD：加载累加器） |
| 12 | PC ← addr（JMP：跳转） |
| 13 | AC<0? PC ← addr（JNG：条件跳转） |

#### 操作示例

1. 进入 Exp12，按 KEY1 复位 CPU

## 7. 故障排除

### 按 KEY0 后系统重启

这是正常行为。KEY0 是整板硬复位，会重启 NEORV32 CPU。重启后需要重新进入 expdemo。

### HEX/LED 显示异常

1. 确认当前 channel 是否正确（VGA Monitor 显示 HW channel）
2. 某些实验的 HEX 显示为低电平有效（段码亮=0）
3. 退出 expdemo（`q` 或 channel=0）后 HEX/LED 归还 shell

### PS/2 键盘无响应（Exp8）

1. 确认 PS/2 键盘已正确连接到 DE2-115 的 PS/2 端口
2. Exp8 独占 PS/2 键盘，此时 shell 的 PS/2 输入不可用
3. 按 `Del` 退出 Exp8 后恢复

### IR 遥控器无响应（Exp10）

1. 确认对准 DE2-115 板载 IrDA 接收器
2. 距离建议在 3 米以内，无遮挡
3. Exp10 独占 IR 输入，MENU 键退出后恢复
4. 检查遥控器电池

### Exp9 无 loopback 数据

1. 确认按了 KEY1 触发发送
2. UART 为内部 loopback（TX 回环到 RX），不需要外部接线
3. VGA Monitor 的 RX History 区域显示累积接收记录

### Exp12 CPU 不执行

1. 先按 KEY1 复位 CPU
2. 手动模式下按 KEY2 单步执行
3. 自动模式下按 KEY3 切换，CPU 持续运行
4. 检查 LCD 和 HEX 是否有输出

### VGA Monitor 不更新

Monitor 仅在输入状态发生变化时刷新（SW、KEY、IR 有变化）。如果没有拨动开关或按键，Monitor 不会频繁刷新。
