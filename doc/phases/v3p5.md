## 1. 概述

ChromaShader 是一个基于 FPGA 硬件噪声网络的彩色终端沙盒世界。核心创意是在 80x25 VGA 文本终端上, 利用硬件实时并行生成无限大的彩色地形世界, PS/2 键盘操控滚动, 零延迟刷新 (BRAM 直读, 全帧重填仅需 ~40us)。

与 UART ANSI 方案相比, VGA 直显方案刷新延迟从 ~0.76s 降低到 < 1ms, 且只需 VGA 显示器和 PS/2 键盘, 无需主机。

### 核心设计

- 时分复用噪声网络: 1 个 LFSR + 坐标散列单元, 2000 周期填满 80x25 全帧
- 地形颜色映射: 水域 (深蓝) / 草地 (绿) / 山地 (棕) / 雪峰 (白) / 金矿 (黄叠加) / 围墙 (灰)
- 双 BRAM 结构: 噪声层 + 改造层 (喷涂颜色覆盖地形)
- 游戏玩法: WASD 滚动, E 采集, 1/2/3 喷涂, 胜利条件为收集 10 个金粒

### 资源估算

| 模块 | LEs | M9K |
|------|-----|-----|
| 噪声生成器 (时分复用) | ~200 | 0 |
| 帧缓存 BRAM (2000x16bit) | 0 | 4 |
| 改造层 BRAM (可选) | 0 | 4 |
| 控制寄存器 + MUX | ~80 | 0 |
| **合计** | **~280** | **8** |

## 2. 状态: 暂缓

ChromaShader 为 V3 最低优先级模块。当前没有分配开发时间。原因:

- V3 核心目标 (SDRAM 执行 + FreeRTOS + PS/2 输入 + VGA 像素模式) 尚未上板验证
- Conway / PONG / NTT 等已完成的模块尚未接入 de2os
- ChromaShader 的 VGA 文本终端直显方案需要评估与 VGA 像素模式是否冲突
- 资源估算约 280 LEs + 8 M9K, 占用很小, 不影响总体资源预算

---

## 3. 简略实现步骤

1. **VHDL 模块编写**: 时分复用噪声网络 (`chroma_noise.vhd`) + 双端口 BRAM + Wishbone 控制寄存器 (基地址待定, 需分配在 0xF0013000 之后), 估算 ~6h
2. **总线接入**: wb_intercon 新增从端口, de2os_top 实例化, VGA 文本终端数据源切换 MUX
3. **C 驱动**: 键盘轮询 + 坐标更新 + 地形采集/喷涂逻辑 + 背包管理, 估算 ~3h
4. **上板验证**: WASD 滚动延迟, 地形颜色连续性, 喷涂持久化, IR 切换 (ChromaShader <-> Conway <-> 正常终端)
