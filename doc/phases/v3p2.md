# V3 Phase 2: ExpDemo 集成到 de2os

## 1. 概述

ExpDemo 是 DE2Extra 的硬件实验演示系统，将 11 个课程实验（通道 1-5, 8-13）以纯 VHDL 硬件形式集成到板卡上，通过统一的 Wishbone 接口（`0xF0010000`）进行通道切换和状态读取。

**目标**: 将 ExpDemo 从 V2 顶层（`de2_115_top.vhd`）移植到 V3 顶层（`de2os_top.vhd`），使得 de2shell_rtos 固件可以通过 `expdemo` 命令在 FreeRTOS 环境下切换和展示全部 11 个硬件实验。

**范围**:
- VHDL 层面：`de2os_top.vhd` 中完整实例化 `expdemo_top` + `expdemo_wb`，连接板级输出路由
- QSF 层面：添加缺失的实验适配器 VHDL 文件、MIF 文件、SW 引脚分配
- C 驱动层面：将 `sw/app/de2shell/demo.c` 移植到 de2shell_rtos

## 3. 架构差异分析

### 3.1 CPU 配置差异

| 项目 | de2_115_top (V2) | de2os_top (V3) |
|------|-----------------|---------------|
| BOOT_MODE | 2（直接 IMEM 执行） | 0（bootloader，SDRAM 执行） |
| ICACHE_EN | true | false |
| ICACHE_BURSTS | true | true（generic 传入但 ICACHE 未使能） |
| IMEM_SIZE | 64KB 固定 | 16KB 默认（可通过 generic 修改） |
| IRQ 集成 | intc_irq -> irq_mei_i | 硬接 '0'（V3 未启用 INTC） |
| xbus_tag_o | open | 传给外部（未连接 wb_intercon，不影响） |

**影响**: ExpDemo 不依赖 CPU 特定配置，所有实验在 50MHz 时钟域运行，与 CPU 配置无关。V3 的 bootloader 和 SDRAM 执行不影响实验硬件。

### 3.2 wb_intercon 差异

| 项目 | de2_115_top (V2) | de2os_top (V3) |
|------|-----------------|---------------|
| s6 | Timer @ 0xF0009000 | BuildInfo @ 0xF0009000 |
| s7 | INTC @ 0xF000A000 | 悬空（open, '0'） |
| s8 | ExpDemo @ 0xF0010000 | ExpDemo @ 0xF0010000 |
| s9 | 无 | PONG @ 0xF0011000 |
| s10 | 无 | Conway @ 0xF0012000 |

**影响**: ExpDemo 地址 `0xF0010000` 在两个顶层中一致，s8 端口行为相同。V3 的 Timer/INTC 地址重映射不影响 ExpDemo。地址解码常量在 `de2extra_pkg.vhd` 中统一管理，无需修改。

### 3.3 外设差异

| 外设 | de2_115_top | de2os_top | ExpDemo 影响 |
|------|------------|-----------|-------------|
| VGA pixel mode | 禁用（stub） | 已实现（`vga_pixel_ctrl`） | 无直接冲突，ExpDemo 不使用 VGA |
| JTAG UART | 有 `uart_jtag_bridge` | 有 `uart_jtag_bridge` | 无冲突 |
| LCD shell 侧 | `lcd_status`（硬件状态显示） | `lcd_wb`（Wishbone 控制器） | 需要处理 LCD 复位恢复 |
| BuildInfo | 无 | `build_info_wb` | 无冲突 |
| PONG/Conway | 无 | stub（已禁用） | 无冲突 |
| NTT | stub | stub | 无冲突 |

### 3.4 引脚所有权差异

de2os_top 与 de2_115_top 共享完全相同的板级引脚。ExpDemo 使用的全部引脚（SW、KEY、HEX0-7、LEDR、LEDG、LCD_DATA/RS/RW/EN、PS2_CLK/DAT、IRDA_RXD、UART_TXD/RXD）在两个顶层中均为相同的物理引脚，不存在引脚冲突。

**关键注意**: de2os QSF 当前缺少 SW[0..15] 引脚分配。ExpDemo 的多个实验依赖这些拨码开关（Exp1 用 SW[5:0]，Exp4 用 SW[17:7:0] 等），缺少引脚分配将导致 SW 输入悬空。

### 3.5 时钟域差异

两个顶层使用相同的时钟方案：
- CPU/XBUS 外设: 50MHz
- SDRAM 控制器: 100MHz
- DRAM_CLK: 100MHz +1.56ns 相移

ExpDemo 全部运行在 50MHz 域，不涉及时钟域交叉问题。

### 步骤 4: de2os_top.vhd -- LCD 退出复位管理

V2 中，从 expdemo 退出回到 shell 模式时，LCD 需要重新初始化（HD44780 需要复位脉冲）。de2_115_top 实现了 `lcd_shell_rst_cnt` 下降计数器（第 373-388 行），在 `expdemo_active` 从 1 变为 0 时触发 50000 周期复位延时。

de2os_top 当前直接使用 `rst_n` 复位 `lcd_wb`（第 770 行），缺少退出 expdemo 时的 LCD 重新初始化逻辑。

修改方案：

1. 添加信号：
   ```vhdl
   signal expdemo_active_d : std_logic := '0';
   signal lcd_shell_rst_cnt : integer range 0 to 50000 := 50000;
   signal lcd_shell_rst_n   : std_logic;
   ```

2. 添加进程：
   ```vhdl
   p_expdemo_lcd_reinit : process (clk_50m, rst_n)
   begin
       if rst_n = '0' then
           expdemo_active_d <= '0';
           lcd_shell_rst_cnt <= 50000;
       elsif rising_edge(clk_50m) then
           expdemo_active_d <= expdemo_active;
           if (expdemo_active_d = '1') and (expdemo_active = '0') then
               lcd_shell_rst_cnt <= 50000;
           elsif lcd_shell_rst_cnt > 0 then
               lcd_shell_rst_cnt <= lcd_shell_rst_cnt - 1;
           end if;
       end if;
   end process;

   lcd_shell_rst_n <= '0' when (rst_n = '0') or (lcd_shell_rst_cnt > 0) else '1';
   ```

3. 修改 `u_lcd` 实例化的复位连接：
   ```vhdl
   rst_n_i  => lcd_shell_rst_n,  -- 原为 rst_n
   ```

## 6. 风险

### 6.1 信号名冲突

de2os_top 已声明大部分 expdemo 相关信号（`expdemo_wb_*`、`exp_hex`、`exp_ledr`、`exp_ledg`、`exp_lcd_*`、`ps2_clk_shell_in`、`ps2_dat_shell_in`、`ps2_clk_exp_in`、`ps2_dat_exp_in`、`irda_shell_in`、`irda_exp_in`、`hex*_shell`）。新增信号仅需 `exp_uart_txd`（步骤 3）和 LCD 复位相关信号（步骤 4），不存在命名冲突。

### 6.2 资源影响

ExpDemo 包含 11 个实验适配器，但每个适配器的资源消耗很小（大部分是纯组合逻辑或小型状态机）。预估增量：
- 逻辑单元 (LE): +500-1000（Simple CPU 最大，约 200-300 LE）
- M9K 块 RAM: +2（Dual-Port RAM 在 Exp4 中使用 1 个，DDS ROM 可能使用 1 个）
- PLL: 无变化
- EP4CE115 总资源: 114,480 LE / 432 M9K，增量占比 < 1%

Cyclone IV E EP4CE115F29C7 资源充裕，不存在资源不足风险。

### 6.3 引脚冲突

de2os QSF 已包含所有板级引脚分配（KEY、UART、PS/2、IR、LEDR、LEDG、HEX0-7、LCD、VGA、SDRAM）。唯一缺失的是 SW[0..15]。ExpDemo 不使用任何 de2os 独有的引脚，不存在冲突。

**注意**: SW 引脚分配必须从 `DE2-115引脚表.xlsx` 或已有的 `de2extra.qsf` 复制，不得自行推测。

### 6.4 时序影响

ExpDemo 的实验逻辑全部在 50MHz 域运行，不引入新的时钟域。最复杂的实验（Simple CPU、DDS）的时序路径已在上电时序分析中覆盖。添加这些模块不应导致时序违例，但建议编译后检查 TimeQuest 报告。

### 6.5 C 驱动兼容性

de2shell_rtos 的 HAL 体系与 de2shell 不同。demo.c 中的 VGA 渲染函数（`vga_puts()`、`vga_clear()`）可能需要适配。如果适配工作量过大，可以先实现 UART-only 的精简版 expdemo 命令（仅支持通道切换和 monitor 输出到串口），VGA 渲染作为后续优化。

### 6.6 Exp9 UART TXD 路由冲突

将 `UART_TXD` 在 shell UART 和 Exp9 UART TXD 之间切换，意味着激活 Exp9 时 shell 的 UART 输出将中断。这与 V2 行为一致（de2_115_top 同样存在此行为），是设计预期的。用户需通过 `q` 退出 Exp9 才能恢复 shell UART 输出。
